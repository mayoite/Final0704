# Docs Index

## Purpose

This folder is the canonical documentation area for working material that still matters to the repo.

It is organized by topic, not by when the file happened to be created.

## Structure

```text
IMPORTANTFILES/docs/
  README.md
  audits/
  deployment/
  diagrams/
  planner/
  review-package-main-site-2026-04-08/
```

## Where To Start

| If you need | Start here |
|---|---|
| Current planner direction | `planner/new/PLANNER-DETAILED-PLAN.md` |
| Planner working summary | `planner/new/PLANNER-PLAN-POST.md` |
| Planner historical/reference context | `planner/README.md` |
| Main-site reviewer handoff bundle | `review-package-main-site-2026-04-08/README.md` |
| Deployment docs | `deployment/README.md` |
| Audit docs | `audits/README.md` |
| Process and flow diagrams | `diagrams/README.md` |

## Rules

| Rule | Meaning |
|---|---|
| `planner/` | Planner-specific docs, current and reference |
| `deployment/` | Runtime and deployment documentation |
| `audits/` | Audit and review style documents |
| `diagrams/` | SVG and flow-oriented artifacts |
| Root of `docs/` | Index files only, not loose topic files |

## Notes

- Generated reports in `07docs/` are useful working artifacts, but they are not the canonical docs tree.
- Current planner authority lives in `planner/new/PLANNER-DETAILED-PLAN.md`.
- Historical planner docs are still kept, but they should be read as reference unless marked current.
- The root review bundle lives in `review-package-main-site-2026-04-08/`.
